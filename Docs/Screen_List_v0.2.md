# Little Paw Town - Screen List v0.2

---

## 1. 문서 목적

본 문서는 `Little Paw Town GDD v0.2` 기준으로, MVP에서 필요한 **화면 목록과 각 화면의 역할**을 정리한 문서다.

목적은 다음과 같다.
- 실제 구현 단위를 화면 중심으로 정리
- 화면 간 플로우와 진입/이탈 구조 명확화
- MVP에 꼭 필요한 화면과 통합 가능한 화면을 구분

---

## 2. 화면 설계 원칙

### 2.1 기본 원칙
- 세션 시작 후 최대한 빠르게 **반려동물 확인** 단계로 진입해야 한다.
- 화면 수를 불필요하게 늘리기보다, **오버레이/모달**로 통합 가능한 부분은 합친다.
- 홈 화면은 단순 메뉴가 아니라, **게임의 감정 허브**여야 한다.
- 홈은 **살아있는 타운을 엿보는 관찰 허브**이고, 본격 이벤트 소비는 장소 화면에서 수행한다.
- 이벤트 화면은 텍스트 뷰어가 아니라 **표정/행동/선택의 중심 화면**이어야 한다.

### 2.2 통합 원칙
아래는 별도 전체 화면보다 오버레이/팝업이 더 적절하다.
- 간단 보상 팝업
- 짧은 교감 액션
- 애정도 상승 표시
- 추억 카드 획득 알림
- 설정 일부

---

## 3. 전체 플로우 요약

```text
스플래시
→ 타이틀
→ 로그인/게스트 진입
→ 튜토리얼(최초 1회)
→ 반려동물 생성
→ 생성 완료 연출
→ 홈
  → 짧은 생활 반응 자동 재생
  → 상태 패널 확인
  → 장소 이동 또는 짧은 상호작용
  → 이벤트
  → 추억 저장/확인
  → 하우징
  → 상점
  → 미니게임
  → 설정
```

## 4.3 코어 허브 영역

### SCR-020 홈
**목적**
- 핵심 허브 화면
- 오늘의 반려동물 상태와 분위기 확인
- 관찰 보상과 즉시 행동 진입을 동시에 제공
- 대부분의 루프가 여기서 시작됨

**화면 구조**
- 상단 요약 바
  - 시간/날씨
  - 애정도 요약
  - 오늘의 분위기 한 줄
- 중앙 관찰 영역
  - 홈 주변 공간 배경
  - 반려동물 자율 행동
  - 짧은 말풍선/감정 아이콘
- 중단 상태 패널
  - 현재 기분/행동 힌트
  - 추천 장소 또는 추천 이벤트 CTA
- 하단 행동 영역
  - 간단 상호작용 버튼
    - 쓰다듬기
    - 간식 주기
    - 반응 보기
  - 장소 이동 버튼
  - 추억/하우징/상점/설정 진입 버튼

**핵심 UX 규칙**
- 홈 진입 후 2~3초 내 짧은 생활 반응 1회를 자동 재생한다.
- 플레이어는 반려동물 이동을 직접 조작하지 않는다.
- 홈에서는 **홈 주변에서 살아가는 느낌**을 보여주고, 공원/베이커리/광장 같은 본격 장소 소비는 별도로 유지한다.
- 홈은 강제 이벤트 팝업보다 **관찰 → 상태 확인 → 선택형 이동/상호작용** 흐름을 우선한다.

**중요도**
- 최상위 P0

**비고**
- 이 화면 품질이 전체 제품 인상을 좌우함
- MVP에서는 레이아웃 단순성을 유지하되 중앙 관찰 연출만 강화하는 방향 권장

---

### CMP-021 애정도 상승 팝업
**목적**
- 애정도 상승 결과를 짧게 체감시킴

**핵심 요소**
- +포인트
- 단계 상승 여부
- 해금 반응/이벤트 안내

**형태**
- 홈/이벤트 결과 오버레이

---

### CMP-022 오늘의 상태 패널
**목적**
- 현재 분위기와 가능 행동을 압축 제시
- 관찰을 실제 행동으로 연결하는 브리지 역할 수행

**핵심 요소**
- 시간대
- 날씨
- 반려동물 현재 기분/행동 힌트
- 추천 장소 또는 이벤트 힌트
- 우선 CTA 1개
- 보조 행동 0~1개

**형태**
- 홈 내 패널

**권장 규칙**
- 홈 진입 직후 자동 노출한다.
- 문구는 "오늘 어디를 가볼지"를 제안하는 수준으로 간결하게 유지한다.
- 본격 이벤트를 바로 강제하지 않고, 유저가 선택해서 진입하게 한다.

---

### CMP-023 홈 반응 말풍선
**목적**
- 짧은 생활 반응을 시각적으로 강조

**핵심 요소**
- 1줄 반응 문구 또는 감정 아이콘
- 탭 시 짧은 보이스/애니메이션 재생
- 자동 사라짐

**형태**
- 홈 중앙 관찰 영역 플로팅 UI

## 8. 화면별 오픈 이슈

### 8.1 홈 화면
- 한 화면에 정보가 너무 많아질 위험 있음
- 애정도, 재화, 행동 버튼, 이동 버튼을 모두 넣으면 복잡해질 수 있음

**권장안**
- 상단: 오늘의 분위기
- 중앙: 반려동물
- 하단: 장소/메타 버튼
- 즉시 상호작용은 1~2개만 노출

### 8.2 생성 화면
- 종 확장 대응을 과하게 넣으면 초기 UX가 길어짐

**권장안**
- 2단계 생성으로 제한
  - 외형
  - 성향

### 8.3 이벤트 화면
- 텍스트 위주면 감성형 게임의 강점이 약해짐

**권장안**
- 한 번에 보여주는 문장 수를 줄이고
- 표정/움직임/간단 컷 연출을 우선

### 8.4 하우징 화면
- 자유 배치형은 조작 비용이 큼

**권장안**
- MVP는 슬롯형 또는 반자유 배치형 고려

## 10. 최종 정리

MVP 화면 설계의 핵심은 화면 수를 많이 만드는 것이 아니라,
**홈 → 장소 → 이벤트 → 추억/애정도 반영** 흐름을 가장 부드럽게 만드는 것이다.

따라서 우선순위는 다음과 같다.
- 홈 품질 확보
- 이벤트 화면 감정 밀도 확보
- 생성 화면 이탈 최소화
- 추억 앨범과 애정도 피드백을 분명하게 제공

이 네 가지가 살아 있으면, 나머지 화면은 비교적 단순해도 MVP로서 기능한다.
