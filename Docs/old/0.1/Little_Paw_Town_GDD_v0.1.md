# Little Paw Town GDD v0.1

---

## 1. 프로젝트 개요

### 1.1 게임 한 줄 소개
현실 시간의 분위기를 따라 변하는 따뜻한 타운에서, 내가 만든 반려동물의 일상을 관찰하고 교감하며 추억과 애정을 쌓는 감성 생활형 반려동물 육성 게임.

### 1.2 장르 / 플랫폼 / 개발 방향
| 항목 | 내용 |
|------|------|
| 장르 | 감성형 반려동물 육성 / 관찰 / 생활 시뮬레이션 |
| 플랫폼 | 모바일 우선 (iOS / Android) |
| 엔진 | Unity |
| 그래픽 스타일 | 현대적 감성 도트 |
| 기본 플레이 감각 | 짧게 접속해도 만족감이 있는 생활형 루프 |

### 1.3 프로젝트 목표
이 게임의 목표는 강한 경쟁이나 자극적 보상 루프를 제공하는 것이 아니다. 플레이어가 **"내 반려동물이 오늘 어떻게 지냈는지"**를 확인하고, 작은 사건과 감정을 쌓아가며 장기적인 애착을 형성하게 만드는 것이다.

### 1.4 제품 비전
- 반려동물을 "관리"하는 게임보다 **함께 살아가는 느낌**을 주는 게임
- 하루 1~2번 짧게 접속해도 만족감이 있는 게임
- 매일의 작은 장면이 **추억, 관계, 애정도, 집 변화**로 누적되는 게임
- 초기에는 제한된 종으로 시작하더라도, 구조적으로는 **고양이·기타 반려동물** 확장이 가능한 게임

### 1.5 검증하고 싶은 핵심 가설
1. 유저는 "내 반려동물을 만드는 과정" 자체에서 애착을 느낀다.
2. 유저는 "오늘 무슨 일이 있었는지 확인하는 루프"에 반복 접속한다.
3. 유저는 추억과 애정도가 시각적으로 축적될 때 장기적으로 머문다.
4. 종이 늘어나더라도 핵심 감정은 **관찰과 교감**에서 유지된다.

## 2. 핵심 경험 정의

### 2.1 핵심 경험 4축
1. **만들기**: 내 반려동물을 직접 만들고 기본 성향을 정한다.
2. **관찰하기**: 오늘의 날씨, 장소, 시간, 종 특성에 따라 달라지는 모습을 본다.
3. **교감하기**: 반응 선택, 짧은 행동, 이벤트 대응을 통해 관계를 만든다.
4. **남기기**: 장면이 추억, 애정도, 관계, 집 꾸미기 요소로 남는다.

### 2.2 이 게임이 주는 감정
- 따뜻함
- 생활감
- 작은 웃음
- 애착
- 잔잔한 기대감

### 2.3 지양하는 방향
- 강한 FOMO
- 경쟁 중심 구조
- 반복 숙제형 일일 미션
- 강한 패널티 기반 돌봄 압박
- 돈 벌기 위한 노동형 플레이 감각

## 3. 타겟 유저

### 3.1 메인 타겟
- 현실적으로 반려동물을 키우기 어려운 사람
- 실제 반려견/반려묘를 닮은 캐릭터를 만들고 싶은 현재 반려인
- 짧은 시간에 감정적 만족을 얻는 생활형 게임을 선호하는 유저

### 3.2 서브 타겟
- 떠나간 반려동물의 추억을 간접적으로 남기고 싶은 사람
- 귀여운 동물 관찰 자체를 좋아하는 사람
- 힐링형 게임을 찾는 유저

### 3.3 타겟 포지셔닝 원칙
- 전면 메시지: **나만의 반려동물 만들기 + 따뜻한 타운 생활**
- 서브 감성 가치: **추억 축적, 위로, 생활감, 애정 형성**
- 마케팅은 슬픔보다 **애착과 일상성** 중심으로 잡는다.

## 4. 차별화 포인트

### 4.1 실제 핵심 차별화
1. **살아있는 반려동물처럼 보이는 일상 반응**
2. **내 반려동물을 직접 만드는 높은 개인화**
3. **감성 도트 기반의 따뜻한 타운 생활 연출**
4. **작은 해프닝이 추억과 애정도로 누적되는 구조**
5. **종이 달라도 유지되는 공통 감정 루프 + 종별 반응 차이**

### 4.2 확장 차별화
- AI 기반 닮은꼴 반려동물 생성
- 시즌/날씨/시간 기반 연출 강화
- 장기적으로 공유 가능한 앨범 시스템
- 고양이 및 기타 반려동물 업데이트를 통한 장기 확장성

### 4.3 정리
초기 버전의 핵심 경쟁력은 AI가 아니라, **"내 반려동물이 타운에서 살아가는 느낌"**이다. 종 확장은 콘텐츠 볼륨 확장이 아니라, 같은 감정 경험을 다양한 반려동물로 넓히는 수단으로 본다.

## 5. 세계관과 톤

### 5.1 세계관 정의
착하고 귀여운 사람과 동물들이 함께 살아가는 따뜻한 타운. 플레이어의 반려동물은 이 타운의 주민 중 하나로 살아가며, 장소와 인물, 날씨와 시간에 따라 매일 다른 장면을 만든다.

### 5.2 톤 원칙
- 모든 장소와 캐릭터는 기본적으로 안전하고 친절하다.
- 개그는 병맛이 아니라 **생활형 / 표정형 / 상황형**으로 간다.
- 반려동물은 사고를 쳐도 밉지 않은 **귀여운 사고뭉치**여야 한다.
- 특정 종만을 전제로 한 표현보다, 종별 차이를 살리되 전체 세계관은 부드럽게 통합한다.

### 5.3 장소 후보
| 장소 | 역할 |
|------|------|
| 홈 | 하루의 시작과 끝, 휴식, 꾸미기 중심 |
| 공원 | 산책, 놀이, 친구 만남 |
| 베이커리 | 간식, 냄새 추적, 생활형 해프닝 |
| 광장 | 주민 교류, 작은 타운 이벤트 |
| 골목 | 탐험, 숨은 발견, 장난스러운 사건 |
| 호수 | 감성 연출, 후기 확장 장소 |
| 포토부스 | 사진/앨범 기능 연계, 후기 확장 |

## 6. 코어 루프

### 6.1 기본 루프
```text
접속
→ 오늘의 타운 분위기 확인
→ 반려동물 상태 / 하고 싶은 것 확인
→ 짧은 행동 또는 장소 선택
→ 이벤트 발생 및 반응 감상
→ 추억 / 애정도 / 관계 / 집 변화 획득
→ 종료
```

### 6.2 세션 단위 설계

#### 짧은 세션 (1~3분)
- 홈에서 반려동물 상태 확인
- 한 번 쓰다듬기 / 간식 주기 / 짧은 반응 선택
- 오늘의 짧은 이벤트 1개 감상
- 애정도 관련 소규모 반응 1회 확인

#### 일반 세션 (5~10분)
- 장소 1~2곳 방문
- 이벤트 2~3개 경험
- 미니게임 1회
- 사진 저장 또는 소품 배치
- 애정도 상승에 연결되는 의미 있는 장면 1회 경험

### 6.3 개입 비율
- 관찰 70 / 개입 30을 기본 원칙으로 한다.
- 다만 관찰만으로 끝나지 않도록, 유저가 최소한 **오늘 어디 갈지 / 무엇을 준비할지 / 어떤 반응을 선택할지** 결정하게 한다.
- 애정도는 단순 탭 반복이 아니라 **어떤 장면에 어떻게 반응했는가**에 의해 주로 누적되어야 한다.

## 7. 메타 진행 구조

### 7.1 메타 진행 4축

#### A. 애정도 성장
- 플레이어와 반려동물 사이의 관계를 나타내는 핵심 축.
- 단순 접속 보상이 아니라 **이벤트 경험, 반응 선택, 함께한 장면**을 통해 오른다.
- 애정도 단계가 오르면 전용 반응, 대사, 홈 행동, 추억 이벤트가 해금된다.

#### B. 관계 성장
- 주민 / 동물 친구와의 친밀도가 오른다.
- 친밀도 단계가 오르면 전용 이벤트, 선물, 대사, 소품이 해금된다.
- 관계는 숫자보다 장면과 기록으로 체감되게 한다.

#### C. 추억 앨범 축적
- 이벤트 결과가 자동으로 추억 카드로 저장된다.
- 특정 추억 세트를 모으면 앨범 프레임, 벽 장식, 특별 문구, 포즈가 해금된다.
- 플레이어는 "오늘의 장면"이 사라지지 않고 남는다는 감각을 얻는다.

#### D. 집과 생활 공간 변화
- 홈은 단순 장식 공간이 아니라 누적 애착의 시각화 공간이다.
- 가구, 장난감, 사진, 기념품이 쌓이며 집 분위기가 변한다.
- 일부 아이템은 반려동물 행동 반응을 바꾼다.

### 7.2 메타 진행 예시
| 누적 요소 | 결과 |
|------|------|
| 애정도 2단계 달성 | 홈에서 먼저 다가오는 행동 해금 |
| 특정 주민과 친밀도 2단계 달성 | 해당 주민 전용 해프닝 해금 |
| 베이커리 관련 추억 5개 수집 | 빵 테마 앨범 장식 해금 |
| 집 소품 10개 배치 | 홈 내 행동 다양성 증가 |

### 7.3 설계 원칙
- 수치보다 **변화가 보이는 결과물**이 중요하다.
- 메타 진행은 성능 강화가 아니라 애착 강화에 가깝다.
- 장기 목표는 있어야 하지만 압박감은 낮아야 한다.
- 애정도는 이 게임의 핵심 축이지만, 반복 노동으로 파밍되는 구조는 피한다.

## 8. 반려동물 생성 시스템

### 8.1 생성 방향
이 게임의 생성 시스템은 특정 종 하나에 고정되지 않고, **종 선택 + 공통 파츠 구조 + 종별 차이 연출**을 전제로 설계한다.

### 8.2 지원 종 방향
| 구분 | 방향 |
|------|------|
| 초기 구현 | 최소 1종 우선 출시 가능 |
| 기본 확장 대상 | 강아지, 고양이 |
| 장기 확장 대상 | 토끼, 햄스터 등 기타 반려동물 검토 |
| 설계 원칙 | 콘텐츠는 점진적으로 추가하되, 데이터 구조는 종 확장을 전제로 설계 |

### 8.3 생성 방식
#### 직접 생성
- 기본 파츠 조합 방식
- 종 선택
- 털색, 귀, 눈, 무늬, 몸집, 꼬리, 소품 조정
- 초기 버전의 메인 방식

#### AI 생성 (후기 추가)
- 실제 반려동물 사진 기반 닮은꼴 캐릭터 생성
- 완전 복제가 아니라 "닮은 느낌" 중심
- 반드시 수동 보정 단계를 포함

### 8.4 종 확장 설계 원칙
- **공통 레이어**: 이름, 성향, 애정도, 추억, 관계, 홈 데이터
- **종별 레이어**: 기본 체형, 파츠 종류, 이동/반응 애니메이션, 전용 이벤트 텍스트
- 이벤트는 가능하면 공통 원형을 유지하고, 종별 반응 문구와 행동만 분기한다.
- UI 문구는 특정 종만 전제하는 표현을 최소화한다.

### 8.5 생성 UX 원칙
- 첫 생성 과정은 3분 내외로 끝나야 한다.
- 너무 많은 선택지를 한 번에 주지 않는다.
- 생성 직후 바로 애착을 느끼게 하는 짧은 애니메이션이 필요하다.
- 종이 늘어나더라도 생성 UX가 과도하게 길어지지 않도록 단계 수를 제한한다.

### 8.6 핵심 메시지
이 게임에서 생성 시스템은 꾸미기 기능이 아니라, **애착 형성의 첫 진입점**이다.

## 9. 반려동물 특성 시스템

### 9.1 특성 구조
기본적으로는 종 공통 특성을 사용하되, 필요시 종별 보조 특성을 추가한다.

| 항목 | 1단계 | 2단계 | 3단계 |
|------|-------|-------|-------|
| 호기심 | 낮음 | 보통 | 높음 |
| 활발도 | 낮음 | 보통 | 높음 |
| 사교성 | 낮음 | 보통 | 높음 |
| 식성 | 까다로움 | 보통 | 왕성함 |
| 경계심 | 높음 | 보통 | 낮음 |

### 9.2 특성 역할
- 이벤트 반응 분기
- 표정 / 행동 빈도 변화
- 돌발 해프닝 성향 변화
- 특정 장소 선호도 간접 반영
- 애정도 상승 조건의 미세 조정

### 9.3 종별 보정 방향
- 강아지: 사교성, 활발도 반응 가시성이 높음
- 고양이: 경계심, 호기심 반응 연출 가시성이 높음
- 기타 종 추가 시, 기존 5개 특성을 최대한 재사용하고 부족할 때만 보조 태그를 추가한다.

### 9.4 설계 원칙
- 이벤트 한 개가 모든 특성을 보지 않는다.
- 보통 1~2개 특성만 체크해 제작 복잡도를 억제한다.
- 유저는 특성을 숫자가 아니라 성격으로 체감해야 한다.

## 10. 애정도 시스템

### 10.1 목적
애정도는 플레이어와 반려동물 사이의 유대감을 표현하는 핵심 수치이자, **이 게임의 장기 체류 동기**를 만드는 장치다. 다만 성능 강화 수치가 아니라, 행동 변화와 감정적 보상으로 체감되어야 한다.

### 10.2 애정도와 친밀도의 차이
| 항목 | 의미 |
|------|------|
| 애정도 | 플레이어 ↔ 반려동물 사이의 유대감 |
| 친밀도 | 반려동물 ↔ 주민/동물 친구 사이의 관계 |

### 10.3 애정도 상승 원천
우선순위는 다음과 같다.
1. **이벤트 완료 및 선택 반응**
2. **특정 상황에서 적절한 돌봄/교감 제공**
3. **첫 방문, 첫 추억, 선호 장소 경험 같은 특별 순간**
4. **가벼운 일상 상호작용**

즉, 애정도는 기본적으로 **이벤트 중심**으로 오르며, 단순 반복 탭만으로 빠르게 올릴 수 없게 설계한다.

### 10.4 애정도 단계 예시
| 단계 | 명칭 | 체감 변화 |
|------|------|-----------|
| 0 | 낯섦 | 기본 반응만 제공 |
| 1 | 익숙함 | 홈 대기 반응 증가 |
| 2 | 편안함 | 먼저 다가오거나 쉬는 행동 다양화 |
| 3 | 신뢰 | 전용 이벤트, 특별 표정, 추억 컷신 해금 |
| 4 | 깊은 애착 | 홈 상호작용 확장, 특별 기념 연출 해금 |

### 10.5 상승 설계 원칙
- 의미 있는 이벤트에서 큰 폭으로 오른다.
- 일상 행동은 보조 수단이며, 반복 효율은 낮다.
- 하루에 과도한 파밍이 일어나지 않도록 완만한 체감 상한 또는 효율 감소를 둔다.
- 애정도는 실패 페널티보다 **함께한 경험의 누적**으로 오르게 한다.

### 10.6 애정도 보상 방향
- 홈에서 먼저 다가오기, 따라다니기, 졸졸 쫓아오기
- 특정 장소에서 반려동물 쪽에서 먼저 제안하는 반응
- 특별한 추억 카드 / 기념 컷 / 앨범 장식 해금
- 일부 전용 이벤트에서 감정 밀도가 높은 후속 장면 해금

### 10.7 주의사항
- 애정도 아이템 직접 판매 금지
- 애정도 하락을 과도한 죄책감 장치로 사용하지 않음
- 애정도는 숙제가 아니라, 플레이어가 자연스럽게 "같이 보낸 시간"으로 느끼게 해야 한다.

## 11. 이벤트 시스템

### 11.1 기본 철학
이벤트는 문장 수를 무한정 늘리는 방식이 아니라, **원형 이벤트 + 조건 분기 + 후속 결과** 구조로 설계한다.

### 11.2 이벤트 분류
| 카테고리 | 설명 |
|------|------|
| 환경 반응 | 날씨, 시간, 장소에 따른 반응 |
| 습관 반응 | 성격과 상태에 따른 일상 행동 |
| 관계 이벤트 | 주민/동물 친구와의 상호작용 |
| 애정도 이벤트 | 플레이어와의 유대가 핵심인 장면 |
| 연쇄 이벤트 | 이전 사건의 후속 장면 |

### 11.3 이벤트 결과 원칙
- 성공/실패보다 감정과 장면이 중요하다.
- 강한 손실 대신 귀여움, 민망함, 뿌듯함, 애틋함을 남긴다.
- 이벤트는 추억 카드나 대화 로그 형태로 기록될 수 있다.
- 주요 이벤트는 애정도 상승과 연결될 수 있어야 한다.

### 11.4 종 확장 대응 원칙
- 원형 이벤트는 최대한 공통으로 유지한다.
- 같은 상황이라도 강아지와 고양이는 **반응 방식**이 달라야 한다.
- 신규 종 추가 시 기존 이벤트를 전부 새로 만드는 대신, 원형 이벤트에 종별 연출 레이어를 추가한다.

### 11.5 현실적인 제작 규모
| 단계 | 원형 이벤트 수 |
|------|---------------|
| 프로토타입 | 20~30개 |
| MVP | 40~50개 |
| 출시 초기 | 80~120개 |

### 11.6 이벤트 작성 원칙
- 텍스트보다 **행동과 표정**으로 기억되게 한다.
- 같은 원형 이벤트도 장소/날씨/성격/종에 따라 다른 느낌이 나야 한다.
- 이벤트 하나가 다음 날 후속 장면으로 이어질 수 있어야 한다.

## 12. 현실 시간 / 날씨 연동

### 12.1 목적
현실 연동은 보상을 강제하기 위한 장치가 아니라, **오늘의 분위기를 다르게 만드는 장치**다.

### 12.2 MVP 반영 범위
- 시간대: 아침 / 낮 / 저녁 / 밤
- 날씨: 맑음 / 비

### 12.3 출시 이후 확장 범위
- 눈
- 계절 변화
- 현실 기념일 연출

### 12.4 원칙
- 특정 시간에 못 들어왔다고 손해를 느끼게 하지 않는다.
- 접속하면 "오늘은 이런 날이구나"를 느끼게 해야 한다.
- 시간/날씨는 애정도 이벤트의 분위기 증폭 장치로 사용할 수 있다.

## 13. 알림형 돌발 이벤트

### 13.1 역할
앱을 꺼둔 사이에도 반려동물이 자기 삶을 살아가는 느낌을 주는 장치.

### 13.2 톤
- 귀엽고 황당한 상황극
- 죄책감 유발이 아니라 웃음과 애정 유발
- 재화 소비는 선택적으로만 연결

### 13.3 운영 가드레일
- 하루 최대 푸시 횟수 제한 필요
- 같은 유형 푸시 반복 쿨다운 필요
- 미접속 유저 대상 푸시는 압박 메시지 금지
- 손실 회피형 문구 금지

### 13.4 예시 방향
- 베이커리 외상 해프닝
- 혼자 장난감 찾다 집안을 어질러 둔 사건
- 친구와 놀다 예상 밖 선물을 받아온 사건
- 플레이어가 반응하면 애정도 후속 장면이 열리는 알림형 이벤트

## 14. 경제 시스템

### 14.1 철학
재화는 생존 압박이 아니라, **생활을 더 풍성하게 만드는 수단**이다.

### 14.2 재화 구조
| 재화 | 역할 |
|------|------|
| 기본 머니 | 장난감, 간식, 기본 꾸미기 구매 |
| 감성 토큰 | 추억/애정도/특별 장면 기반 해금용 |

### 14.3 사용 방향
- 기본 돌봄은 사실상 무료에 가깝게 유지
- 돈이 없어서 반려동물을 방치하는 느낌은 피한다.
- 소비는 집 꾸미기, 사진 장식, 특별 준비 같은 애정 표현으로 연결한다.

### 14.4 주의사항
- 애정도 직접 구매 금지
- 종 추가를 지나치게 과금 게이트화하지 않도록 주의
- 돌봄 필수 행동은 과금과 분리

## 15. 미니게임

### 15.1 역할
미니게임은 메인이 아니라 보조 루프다. 반려동물과 타운 생활을 더 풍성하게 만드는 수단으로만 사용한다.

### 15.2 적합한 방향
- 공놀이
- 냄새 추적
- 간식 굽기
- 숨은 장난감 찾기
- 종별 차이를 살릴 수 있는 가벼운 놀이

### 15.3 지양하는 방향
- 경쟁 랭킹 중심
- 과한 반응속도 요구
- 반복 실패 스트레스
- 전투형 시스템

### 15.4 애정도와의 연결 원칙
- 미니게임은 애정도 보조 상승 수단으로 사용 가능
- 단, 미니게임 반복 플레이가 가장 효율적인 애정도 파밍 루프가 되어서는 안 된다.

## 16. BM 방향

### 16.1 권장 모델
**무료 다운로드 + 선택형 코스메틱 인앱 과금**을 기본 가정으로 한다.

### 16.2 이유
- 감성 생활형 게임은 초반 진입장벽이 낮을수록 유리하다.
- 유저가 애착을 먼저 형성한 뒤 꾸미기 과금이 자연스럽게 이어진다.
- 진행 우위 판매 없이도 게임 톤을 유지할 수 있다.

### 16.3 과금 대상
- 반려동물 의상
- 하우징 테마 세트
- 앨범 프레임 / 스티커
- 시즌 한정 소품
- 향후 종별 코스메틱 팩

### 16.4 금지 원칙
- 반려동물 돌봄 기능 유료화 금지
- 핵심 이벤트 접근 유료화 금지
- 성능 우위 판매 금지
- 애정도 직접 판매 금지

## 17. MVP 정의

### 17.1 MVP 목표
다음을 검증한다.
1. 내 반려동물을 만드는 재미가 있는가
2. 오늘의 장면을 확인하러 다시 들어오게 되는가
3. 추억과 애정도가 쌓일 때 애착이 강화되는가
4. 종 확장을 고려한 구조가 초기 개발 효율을 해치지 않는가

### 17.2 MVP 포함 범위
| 항목 | 범위 |
|------|------|
| 반려동물 생성 | 직접 생성만 |
| 지원 종 | 최소 1종 우선 구현, 구조는 강아지/고양이 확장 대응 |
| 외형 파츠 | 최소 조합형 파츠 세트 |
| 특성 시스템 | 공통 5개 특성 |
| 애정도 시스템 | 5단계 구조 + 이벤트 중심 상승 |
| 장소 | 홈, 공원, 베이커리, 광장 |
| 이벤트 원형 | 40~50개 |
| 시간/날씨 | 시간대 4단계 + 맑음/비 |
| 미니게임 | 1~2종 |
| 관계 캐릭터 | 주민 2명 + 동물 친구 1마리 |
| 하우징 | 기본 소품 8~12종 |
| 추억 시스템 | 간단한 추억 카드 저장 기능 |
| 돌발 알림 | 3~5종 |
| 경제 시스템 | 기본 머니 중심 |

### 17.3 MVP 미포함
- AI 반려동물 생성
- 계절 시스템
- 감성 토큰 고도화
- 포토부스 / 호수 / 리듬하우스
- GPS 기반 소셜
- 다두/다묘/복수 반려동물 동시 육성
- 기타 반려동물 종 본격 추가

### 17.4 MVP 성공 기준 예시
- D1 / D7 리텐션 확인
- 1인당 생성 완료율
- 하루 평균 세션 수
- 추억 카드 저장/열람 비율
- 홈 꾸미기 사용률
- 애정도 2단계 이상 도달 비율

## 20. 최종 요약

Little Paw Town 은 강한 경쟁이나 관리 압박 대신, **내 반려동물의 오늘을 확인하고 그 장면을 추억과 애정으로 쌓는 게임**이다.

이 게임의 핵심은 다음 네 가지다.
1. 내 반려동물을 만들고 애착을 느끼는 것
2. 매일 다른 작은 장면을 관찰하고 반응하는 것
3. 그 장면이 애정도, 관계, 추억, 집 변화로 누적되는 것
4. 초기에는 작은 범위로 시작하되, 장기적으로는 다양한 반려동물로 확장 가능한 구조를 갖추는 것

초기 버전에서는 기능을 넓게 펼치기보다, 이 네 가지가 실제로 재미와 리텐션을 만드는지 먼저 검증하는 것이 중요하다.
